Project summary

Spring Siege Event — mock campaign

For this project, I imagined a fictional seasonal in-game event for a live mobile strategy game. The goal was to build a practical support system that helps a marketing team plan launches, keep assets organized, track deadlines, and make sure player communication is timely and clear.

Workflow

How I structured the campaign process

I wanted the workflow to be simple, practical, and realistic for a fast-moving team. The structure below shows how planning, coordination, launch support, and follow-up fit together.

01
Campaign Brief
Define event theme, goals, channels, deliverables, and what players need to know.
02
Tracker Setup
Organize deadlines, owners, approvals, blockers, asset status, and publishing flow.
03
Content Planning
Map teasers, launch posts, reminders, and community updates into a clear calendar.
04
Launch & Review
Monitor execution, gather notes, summarize what shipped, and capture learnings for the next cycle.
